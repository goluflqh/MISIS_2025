О проекте

Что мы будем делать?

Полноценный веб-сервис с аналитикой, ML-моделями и автоматическим обновлением данных

01

📥 Собираем данные

Пишем скрипты для автоматического сбора данных из API и веб-страниц. Данные сохраняются в PostgreSQL и обновляются по расписанию.

02

🔬 Анализируем и моделируем

Проводим EDA, строим гипотезы, создаём признаки и обучаем ML-модели. От простой регрессии до NLP и временных рядов.

03

🌐 Создаём веб-сервис

FastAPI для REST API с эндпоинтами данных и предсказаний. Streamlit для интерактивного дашборда с графиками и фильтрами.

04

🐳 Упаковываем в Docker

Весь проект разворачивается одной командой docker compose up. База, API, сборщик данных, дашборд — всё в контейнерах.

💡 Главная идея

Каждый студент выбирает свой домен (вакансии, недвижимость, крипто, игры...), но использует единую архитектуру. Это позволяет иметь разнообразие тем при единых критериях оценки.

services:
  db:
    image: postgres:16-alpine
    environment:
      POSTGRES_DB: datapulse
      POSTGRES_USER: ${DB_USER}
      POSTGRES_PASSWORD: ${DB_PASSWORD}
    volumes:
      - postgres_data:/var/lib/postgresql/data

  redis:
    image: redis:7-alpine

  ingestor:
    build: ./services/ingestor
    depends_on: [db]
    environment:
      DATABASE_URL: postgresql://${DB_USER}:${DB_PASSWORD}@db:5432/datapulse

  api:
    build: ./services/api
    ports: ["8000:8000"]
    depends_on: [db]

  dashboard:
    build: ./services/dashboard
    ports: ["8501:8501"]
    depends_on: [api]

Критерии оценки

Что нужно сделать на минимальную оценку и как получить максимум

4
Хорошо — Обязательный минимум
  • ✓ Воспроизводимость: docker compose up --build запускает проект
  • ✓ Данные: ≥2 источника, ≥1000 записей, обработка пропусков
  • ✓ EDA: Jupyter notebook с визуализациями и выводами
  • ✓ ML: Baseline модель + корректная валидация + метрики лучше naive
  • ✓ API: FastAPI с /predict, Swagger работает
  • ✓ UI: Streamlit с ≥3 страницами
  • ✓ Документация: README с инструкцией по запуску
5
Отлично — Продвинутый уровень
  • ✓ Всё из "4" плюс:
  • ✓ ML Ops: MLflow для трекинга экспериментов
  • ✓ Интерпретация: SHAP / feature importance в UI
  • ✓ Мониторинг: Страница статуса источников + логи
  • ✓ Тесты: pytest, покрытие ≥50%
  • ✓ CI/CD: GitHub Actions (линт + тесты)
  • ✓ Миграции: Alembic для управления схемой БД
  • ✓ Кэш: Redis для оптимизации запросов
  • ✓ Качество кода: ruff, pre-commit, типизация

FAQ

Частые вопросы

🤔 Можно ли работать в команде? ▼
Да, можно работать в командах до 2-3 человек. В этом случае объём работы должен быть пропорционально больше: больше источников данных, больше ML-моделей, более сложный UI. Каждый участник должен чётко понимать свою часть проекта и уметь её объяснить на защите.
💡 Можно ли взять свою тему? ▼
Да! Свою тему нужно согласовать с преподавателем. Главное условие: должны быть доступные источники данных (API или возможность парсинга) и осмысленная ML-задача. За оригинальную тему можно получить бонусные баллы.
⚠️ Что делать, если сайт блокирует парсинг? ▼

Несколько вариантов:

  • Использовать официальный API (если есть)
  • Добавить задержки между запросами (time.sleep)
  • Использовать ротацию User-Agent
  • Переключиться на альтернативный источник
  • Использовать готовые датасеты с Kaggle как backup
🐳 Нужно ли деплоить проект в облако? ▼
Нет, это не обязательно. Достаточно, чтобы проект запускался локально через Docker. Деплой на внешний сервер (Railway, Render, VPS) — это бонус для дополнительных баллов.
📊 Какую ML-модель использовать? ▼

Начните с простых моделей:

  • Регрессия: LinearRegression → Ridge → RandomForest → CatBoost
  • Классификация: LogisticRegression → RandomForest → CatBoost
  • Временные ряды: Naive baseline → ARIMA → Prophet
  • NLP: TF-IDF + LogReg → TextBlob → transformers

Главное — корректная валидация и сравнение с baseline!

📅 Что если не успеваю к дедлайну? ▼
Лучше сдать MVP (работающий минимум), чем ничего. Приоритеты:
  1. Данные собираются и сохраняются в БД
  2. Есть хоть какая-то ML модель
  3. Streamlit показывает данные
  4. Docker запускается

Красивый UI и продвинутые фичи можно добавить позже.
